Web Server?

2022/01/29

by Jun Sik

The definition of MDN for Web Server is as follows.

On the hardware side, a web server is a computer that stores website component files.
(Component files include HTML documents, images, CSS stylesheets, and JavaScript files)
and distribute these files to end-user devices. The web server is connected to the Internet
and can be accessed through a domain name such as mozilla.org.
On the software side, the web server basically manages how web users access host files.
In this article, the web server is limited to HTTP servers.
The HTTP server is part of the URL (Web addresses)
and HTTP (the protocol your browser uses to display web pages).

DBMS (Database Magnagement System)

2022/02/11

by Jun Sik

In the absence of DBMS, the database, or DB, was directly manipulated by users and applications (APPs).
However, this will not only not guarantee the stability of the database,
but will also increase the logic required to create applications.
And the integrity of the data becomes difficult to keep. DBMS has emerged due to these problems.
The DBMS is directly connected to the database and performs several tasks.
In this process, applications or applications are linked to DBMS rather than databases
to modify, delete, or generate data in the DB. Through this, we can grant 'authorities'
and solve the difficulties of processing concurrent users.

Middleware

2022/02/18

by Jun Sik

With the advent of DBMS, many things have been convenient.
However, there were many disadvantages to the direct connection between the DBMS and the client.
When the DBMS logic changed, the client had to modify them all,
so there were problems with re-deployment of the client, and problems with security
because the logic was included in the client and distributed. The concept that emerged due to this is "middleware."
All business logic is handled by middleware.
This makes it possible for the client to send a request to the server
and display only the response of the server on the screen.
That's why clients no longer need business logic.

WAS (Web Application Server)

2022/03/02

by Jun Sik

WAS is one of the middleware mentioned above.
Usually, web servers only provide static data.
However, the need for dynamic work such as accessing and
processing DBMS has increased, and WAS, which plays this role, has emerged.
Usually, if you install only WAS, the web server functionality is built in,
so it works well without any problems. However, in the field,
the web server and WAS are separated.
This is because if there is a problem with the "failover",
only that part of the connection should be blocked so that the existing server can continue to run.
